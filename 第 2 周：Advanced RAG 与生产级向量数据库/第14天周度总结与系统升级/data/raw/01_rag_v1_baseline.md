# 第一周基础 RAG 系统

第一周的基础 RAG 系统一般由文档加载、文本切分、Embedding、向量检索和 LLM 生成组成。

典型链路是：用户问题先被转换为 query embedding，然后在本地向量库中执行 TopK 相似度搜索，最后把检索到的 chunk 拼接到 prompt 中交给大模型回答。

这个版本的优点是简单、容易理解、容易跑通。但它也有明显短板：只依赖向量检索，无法稳定处理精确术语、参数名、错误码、函数名；没有 BM25 稀疏检索；没有 reranker 精排；没有 retrieval trace；也缺少 Recall@K、MRR 等评估指标。

当知识库变大、文档类型变复杂、用户问题变长或变口语化时，基础 RAG 容易出现召回不足、上下文噪声高、答案没有引用来源、无法定位错误环节等问题。
