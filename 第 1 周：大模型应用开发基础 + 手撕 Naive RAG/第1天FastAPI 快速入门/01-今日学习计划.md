# 第 1 天：FastAPI 快速入门学习计划

> 今日主题：FastAPI 快速入门  
> 参考资料：FastAPI 官方 Tutorial，https://fastapi.tiangolo.com/tutorial/  
> 今日目标：掌握 FastAPI 基础，能够创建路由、处理路径参数、查询参数和请求体，并能启动一个最小可用 API 服务。

## 1. 今天要解决什么问题

今天不是为了把 FastAPI 的所有功能一次学完，而是建立一个清晰的后端 API 入门框架：

1. FastAPI 项目最小结构是什么。
2. `FastAPI()` 应用对象是什么。
3. 路由装饰器 `@app.get()`、`@app.post()` 是如何工作的。
4. 请求如何进入函数参数。
5. 路径参数、查询参数、请求体分别适合处理什么数据。
6. Pydantic 模型为什么能帮我们做校验和文档生成。
7. 如何通过 Swagger UI 验证接口是否能正常工作。

学完后，你应该可以独立写出一个简单的商品 API：

1. `GET /`：返回欢迎信息。
2. `GET /health`：返回服务健康状态。
3. `GET /items/{item_id}`：根据路径参数读取单个商品。
4. `GET /items?skip=0&limit=10`：通过查询参数分页读取商品列表。
5. `POST /items`：通过 JSON 请求体创建商品。
6. `PUT /items/{item_id}`：同时使用路径参数、查询参数和请求体更新商品。
7. `DELETE /items/{item_id}`：根据路径参数删除商品。

## 2. 今日学习总目标

完成今天内容后，你需要达到以下水平：

1. 能说清 FastAPI 是什么，以及它适合什么类型的项目。
2. 能安装 `fastapi` 和 `uvicorn`。
3. 能创建 `main.py`，并写出一个最小 FastAPI 应用。
4. 能使用 `uvicorn main:app --reload` 启动服务。
5. 能访问 `http://127.0.0.1:8000/docs` 查看自动生成的接口文档。
6. 能使用 `@app.get()` 创建 GET 路由。
7. 能使用 `@app.post()` 创建 POST 路由。
8. 能使用路径参数接收 URL 中的一部分，例如 `/items/1`。
9. 能使用查询参数接收 URL 问号后的参数，例如 `/items?skip=0&limit=10`。
10. 能使用 Pydantic `BaseModel` 接收 JSON 请求体。
11. 能理解 FastAPI 自动校验和自动文档的基础原理。
12. 能完成一个小型 CRUD 风格 API 的雏形。

## 3. 今日建议时间安排

### 阶段一：建立整体印象，20 分钟

目标：知道 FastAPI 的定位和今天的学习边界。

学习内容：

1. FastAPI 是一个用于构建 API 的 Python Web 框架。
2. FastAPI 基于 Python 类型提示工作。
3. FastAPI 自动生成 OpenAPI 文档。
4. FastAPI 常用于后端服务、AI 应用接口、RAG 系统 API、内部工具接口。
5. 今天只学基础请求处理，不深入数据库、认证、中间件和部署。

完成标准：

1. 你能解释 API 是前端、客户端或其他系统调用后端能力的入口。
2. 你能解释路由就是“URL + HTTP 方法 + 处理函数”的绑定关系。
3. 你能解释为什么 AI 应用常常需要 FastAPI：把模型能力包装成可调用服务。

### 阶段二：环境准备与最小应用，30 分钟

目标：跑起来第一个 FastAPI 服务。

命令：

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install fastapi uvicorn
python -m uvicorn main:app --reload
```

完成标准：

1. 浏览器访问 `http://127.0.0.1:8000/` 能看到 JSON。
2. 浏览器访问 `http://127.0.0.1:8000/docs` 能看到 Swagger UI。
3. 修改代码后，服务能自动 reload。

### 阶段三：掌握 GET 路由，40 分钟

目标：能创建读取类 API。

必须掌握：

1. `@app.get("/")`
2. `@app.get("/items/{item_id}")`
3. 路径参数类型声明：`item_id: int`
4. 查询参数默认值：`skip: int = 0`
5. 查询参数可选值：`q: str | None = None`

完成标准：

1. 访问 `/items/1` 时，`item_id` 是整数。
2. 访问 `/items/abc` 时，FastAPI 自动返回校验错误。
3. 访问 `/items?skip=10&limit=5` 时，函数收到查询参数。
4. 你能解释路径参数和查询参数的区别。

### 阶段四：掌握 POST 请求体，50 分钟

目标：能接收 JSON 请求体并自动校验。

必须掌握：

1. Pydantic `BaseModel`
2. 请求体模型 `Item`
3. 字段类型声明
4. 可选字段
5. POST 路由
6. Swagger UI 中发送 JSON 请求

完成标准：

1. 你能通过 `/docs` 发送 POST 请求。
2. 请求体缺少必填字段时，FastAPI 自动返回错误。
3. `price` 传入错误类型时，FastAPI 能校验。
4. 你能解释 `BaseModel` 的作用。

### 阶段五：组合参数，40 分钟

目标：能在一个接口中同时使用路径参数、查询参数和请求体。

示例：

```python
@app.put("/items/{item_id}")
def update_item(item_id: int, item: Item, notify: bool = False):
    return {
        "item_id": item_id,
        "notify": notify,
        "item": item,
    }
```

完成标准：

1. 你能解释 FastAPI 如何判断一个参数来自哪里。
2. 你能解释为什么 `item_id` 来自路径。
3. 你能解释为什么 `notify` 来自查询参数。
4. 你能解释为什么 `item: Item` 来自请求体。

### 阶段六：今日小项目，60 分钟

目标：完成一个内存版商品 API。

必须实现：

1. `GET /health`
2. `GET /items`
3. `GET /items/{item_id}`
4. `POST /items`
5. `PUT /items/{item_id}`
6. `DELETE /items/{item_id}`

完成标准：

1. 每个接口都能在 `/docs` 中测试。
2. 每个接口返回清晰 JSON。
3. 找不到商品时返回合理错误。
4. 创建商品时有请求体校验。
5. 你能口头解释每个接口的请求来源和返回结果。
